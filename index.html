Includes basic tests and a GitHub Actions workflow for auto‑deploy to GitHub Pages. ## Local dev ```bash npm install npm run dev ``` ## Tests ```bash npm test ``` ## Deploy to GitHub Pages 1. Create a new GitHub repo and push this code to the `main` branch. 2. Go to **Settings → Pages** and set **Source** to "GitHub Actions" (it will detect the included workflow). 3. Push any commit to `main`. The workflow builds with `VITE_BASE=//` and deploys to Pages automatically. 4. Your site will be live at `https://.github.io//`. ### Custom domain (optional) - In **Settings → Pages**, add your custom domain. - Create a `CNAME` record at your DNS pointing to `.github.io`. ## Customize - Update contact links in `src/App.jsx` (`mailto:`, LinkedIn, GitHub). - Replace resume buttons with your URL or add a `public/resume.pdf` and set the href to `/resume.pdf`. - To add blog posts or more sections, create new components or expand data arrays. ## Notes - This template avoids shadcn/ui to keep the repo minimal — all UI is Tailwind classes. - Icons are from `lucide-react`. - If your repo name changes later, the Pages workflow automatically sets `VITE_BASE` so assets resolve correctly.
